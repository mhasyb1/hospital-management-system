# Contributing to Hospital Management System

## How to Contribute

This document provides guidelines for contributing to the Hospital Management System project.

### Getting Started

1. Clone the repository
2. Create a new branch for your feature/fix: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Commit your changes with clear messages: `git commit -m "Add feature: description"`
5. Push to your branch: `git push origin feature/your-feature-name`
6. Create a Pull Request

### Branch Naming Convention

- `feature/` - for new features
- `bugfix/` - for bug fixes
- `docs/` - for documentation updates
- `refactor/` - for code refactoring

### Commit Message Format

```
[Type] Brief description

Detailed explanation if needed
```

**Types:**
- `[Feature]` - New feature
- `[Fix]` - Bug fix
- `[Docs]` - Documentation
- `[Refactor]` - Code refactoring
- `[Test]` - Test-related changes
- `[Chore]` - Maintenance tasks

### Code Style

- Use consistent naming conventions
- Write meaningful variable and function names
- Add comments for complex logic
- Follow the existing code structure

### Testing

Before submitting a pull request:
1. Test your changes thoroughly
2. Ensure no existing functionality is broken
3. Add test cases if applicable

### Documentation

- Update README.md if adding new features
- Add comments for complex code sections
- Update API documentation if applicable

### Pull Request Process

1. Ensure your PR has a clear title and description
2. Reference related issues (e.g., "Closes #123")
3. Wait for code review before merging
4. Address all review comments

### Questions?

Feel free to create an issue or contact the project maintainers.
